Flujos de Trabajo Principales

Proceso de Venta

1
Iniciar Venta

Usuario (Cajero/Admin) accede al módulo de ventas y selecciona productos

POST /sales
2
Validar Datos

Sistema valida que los productos existan y las cantidades sean mayores a 0

Validación
3
Calcular Total

Se calcula el total sumando todos los lineTotal de los ítems

Cálculo
4
Crear Transacción

Se inicia una transacción de base de datos para crear la Sale y sus SaleItems

DB Transaction
5
Actualizar Inventario

Se resta la cantidad vendida del stock de cada Product

Stock Update
6
Generar Factura

Se llama al InvoiceService para crear un archivo Excel con los datos de la venta

Excel Generation

Proceso de Autenticación

1
Login

Usuario envía credenciales a POST /auth/login

2
Verificar Credenciales

AuthService comprueba email y contraseña con bcrypt

3
Generar Token

Si es válido, se genera un JWT con id, role y name del usuario

4
Almacenar Token

Frontend almacena el token para futuras peticiones

5
Validar Acceso

Para rutas protegidas, AuthGuard valida el token en las cabeceras

6
Verificar Roles

RolesGuard comprueba si el usuario tiene permisos para la acción

Control de Accesos por Rol
Administrador (ADMIN)
  • Gestión completa de usuarios
  • Crear y editar clientes
  • Acceso a todos los reportes
  • Configuración del sistema
Mostrador (MOSTRADOR)
  • Realizar ventas
  • Consultar productos
  • Ver inventario
  • Acceso limitado a reportes

Proceso de Carga de Datos (Seeding)

Catalog Seed

Llena la base de datos con el catálogo inicial de productos (Res, Cerdo) y sus subtipos (cortes)

catalog_seed.ts
User Seed

Crea usuarios por defecto (admins, cajeros) con contraseñas hasheadas

user_extra_seed.ts
History Seed

Genera 120 días de historial de ventas falsas para pruebas de reportes

history_seed.ts
